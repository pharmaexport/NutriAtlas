## No data loss rule

NutriAtlas must not discard any source column.

The original Excel file remains the canonical source. The import pipeline may create normalized tables, but it must preserve:

- the full raw row;
- the CIQUAL source column name;
- the source food code `alim_code`;
- the dataset version;
- the file checksum;
- the source worksheet.

## Primary key

`alim_code` is the official CIQUAL food identifier and maps to `foods.source_food_code`.

The unique food key in NutriAtlas is:

```txt
(dataset_version_id, source_food_code)
```

## Nutrient and constituent import rule

Columns 10 to 83 map to `nutrients` and `food_nutrients` using the `codes INFOODS` worksheet when available.

Column 84, `Facteur de Jones`, is present in the composition worksheet but not listed in the INFOODS worksheet. It must still be preserved as a CIQUAL-specific constituent or parameter.
